üß† What Are Plugins in Semantic Kernel?

In Semantic Kernel, plugins are classes containing functions (also known as ‚Äúskills‚Äù) that can be exposed to a Large Language Model (LLM) like GPT-4.

These functions are:

  • Clearly named
  • Self-describing (via [Description] attribute)
  • Callable by the LLM when needed

Think of a plugin like a toolbox. Each function is a tool inside the box, and the LLM can pick the right tool when solving a user request.

üì¶ Why Use Plugins?

  • ‚úÖ Securely expose code logic to LLMs
  • ‚úÖ Break logic into reusable modules
  • ‚úÖ Auto-discoverable via attributes
  • ‚úÖ Chain multiple functions with planners
  • ‚úÖ Testable and maintainable

using Microsoft.SemanticKernel;
using System.ComponentModel;

public class MathPlugin
{
    [KernelFunction("add")]
    [Description("Adds two numbers.")]
    public int Add(int a, int b) => a + b;

    [KernelFunction("square")]
    [Description("Returns the square of a number.")]
    public int Square(int x) => x * x;
}
